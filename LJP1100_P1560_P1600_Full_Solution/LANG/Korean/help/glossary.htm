게이트웨이
게이트웨이는 유사하지 않은 두 개의 네트워크(예: 다른 통신 프로토콜, 데이터 서식 지정, 구조, 또는 아키텍처)를 연결하는 라우터 또는 컴퓨터입니다. 네트워크가 서브넷으로 나누어지면 게이트웨이는 네트워크 통신량을 제어하기 위해 한 서브넷을 다른 서브넷과 분리하는 데 종종 사용됩니다.
TCP/IP 네트워크에서 각 네트워크 장치는 종종 가장 가까운 게이트웨이의 IP 주소로 구성됩니다. 네트워크에 게이트웨이가 없으면 일반적으로 컴퓨터의 IP 주소가 사용됩니다.
자세한 내용은 대형 IP 네트워크에 프린터 추가를 참고하십시오.
무선 네트워크에 대한 내용은 WAP(무선 액세스 포인트)를 참조하십시오.

서브넷
공통의 주소 구성 요소를 공유하는 네트워크의 일부분입니다. TCP/IP 네트워크 상에서 서브넷은 IP 주소가 모두 동일한 접두 문자를 가진 장치로 정의됩니다. 예를 들어, 100.100.100.으로 시작하는 IP 주소를 가진 모든 장치는 동일한 서브넷의 일부입니다. 네트워크를 서브넷으로 분할하는 것은 보안과 성능상의 이유로 매우 유용합니다. 서브넷 마스크를 사용하여 IP 네트워크를 분할할 수 있습니다.

서브넷 마스크
서브넷 마스크는 TCP/IP 네트워크를 서브넷으로 분할하는 데 사용되는 툴입니다. 서브넷을 구성하면 대형 네트워크를 보다 효율적으로 사용할 수 있습니다. 서브넷 마스크의 형식은 IP 주소(n.n.n.n)와 유사합니다. 여기서 n은 0에서 255까지의 숫자입니다. 서브넷 마스크가 지정되지 않으면 일반적으로 컴퓨터에 지정된 서브넷 마스크가 사용됩니다.
IP 주소에 적용되는 경우 서브넷 마스크는 네트워크/서브넷을 식별하는 데 사용되는 IP 주소의 부분과 개별 호스트 노드를 식별하는 데 사용되는 부분을 지정합니다.

암호문
WPA(Wi-Fi 보호 액세스) 네트워크 생성에 사용되는 일련의 문자열입니다.

이름 지정 방식, 공유 이름
Microsoft의 권장 사항에 의거하여 이름에 사용하는 문자는 7비트 ASCII (최초 127개) 문자로 제한됩니다. 공백은 사용할 수 없습니다. 다음 지침을 따르십시오.

  • 문자 A ~ Z(대문자) 사용
  • 문자 a ~ z(소문자) 사용
  • 0부터 9까지의 숫자
  • 제한되는 특수 문자: $ % ' - _ @ ~ ` ! ( ) { } . ^ # & + , ; = [ ]

이름 지정 방식, 프린터
프린터(또는 일체형 장치)의 이름을 지정할 때는 다음 지침에 따르십시오.

  • 길이가 31자를 초과할 수 없음
  • 문자 A ~ Z(대문자) 사용
  • 문자 a ~ z(소문자) 사용
  • 0부터 9까지의 숫자
  • ASCII 값이 32 미만이거나 127을 초과하는 문자들은 사용하지 마십시오.
  • 쉼표(,)는 사용하지 마십시오.

인증
인증은 무선 네트워크 보안 전략입니다. 인증 체계를 갖춘 네트워크 상의 장치는 공유 키를 암호로 사용하여 해당 키를 알고 있는 장치에 대해서만 통신을 수행합니다. WEP와 달리 인증은 무선 장치 간에 전송된 데이터를 암호화하지 않습니다. 그러나 WEP와 함께 인증을 사용할 수 있습니다. 인증 키와 WEP 키는 동일해도 무방합니다.

    • 이 마법사는 Windows XP, Windows NT 4.0 또는 Windows 2000 에서만 프린터 공유 기능을 지원합니다.
    • 프린터를 공유할 시스템에서 서버 서비스를 사용할 수 있어야 합니다.

콜드 리셋
콜드 리셋을 수행하기 전에는 프린터와 Jetdirect 프린트 서버에 대한 테스트 페이지를 인쇄하는 것이 좋습니다.

HP Jetdirect 외장 프린트 서버를 기본 설정으로 복원하려면 전원 코드를 뽑은 다음 프린트 서버의 테스트 단추를 누른 상태에서 해당 전원 코드를 다시 연결합니다. 그런 다음, 프린트 서버가 다시 초기화될 때까지 기다리십시오.

HP Jetdirect 내장형 프린트 서버를 재설정하려면 프린터(또는 일체형 장치)를 콜드 리셋한 다음 프린트 서버가 다시 초기화될 때까지 기다립니다. (대부분의 HP LaserJet 프린터 모델의 경우 전원을 켠 상태에서 프린터 제어판의 진행 단추를 누르십시오. 그 외 프린터의 경우에는 프린터 설명서를 참조하십시오.)

주의: 프린터를 다시 설정하면 프린터 설정이 모두 지워집니다.

클라이언트와 서버간 인쇄

포트 이름
포트 이름 대화 상자는 Windows NT 와 Windows 2000에서만 나타납니다. 이 대화 상자는 네트워크 통신 모드 설정 대화상자에서 "기본 포트 이름 사용" 이 선택되지 않았거나 IP 통신 모드가 선택된 경우에만 나타납니다.

설치할 프린터에 대해 작성하려는 네트워크 포트 이름을 지정할 수 있습니다. 포트 이름은 프린터 등록 정보에서 포트를 식별하기 위해 사용됩니다.

유효한 포트 이름은 최대 63자 길이로 모든 문자를 사용할 수 있습니다. 포트 이름은 PC에서 고유해야 합니다. 요약 화면과 테스트 페이지에 표시됩니다.

프린터 공유
프린터 공유 기능을 사용하면 다른 네트워크 사용자가 해당 컴퓨터를 통해서 해당 프린터로 인쇄할 수 있습니다. 이 설치 유틸리티는 Windows 2000과 Windows NT 4.0에서만 프린터 공유를 지원합니다. 프린터를 공유할 시스템에서 서버 서비스를 사용할 수 있어야 합니다. 다음 선택사항 중 하나를 선택해야 합니다.

  • 공유 안 함: 컴퓨터를 통해 프린터에 액세스하는 다른 네트워크 사용자를 차단하려면 이 옵션을 선택하여 피어 투 피어 인쇄 방식으로 컴퓨터를 설정하십시오.
  • 공유: 이 옵션을 선택하여 컴퓨터를 클라이언트-서버 인쇄에 대한 서버로 설정할 수 있습니다. 네트워크 클라이언트는 해당 컴퓨터를 통해 프린터에 인쇄 작업을 지시할 수 있습니다.

프린터 드라이버
프린터를 제어하는 프로그램입니다. 문서를 인쇄할 때마다 프린터 드라이버는 정확한 제어 명령으로 데이터를 프린터에 전송합니다. 요즘 대부분의 운영 체제에는 가장 일반적인 유형의 프린터용 프린터 드라이버가 들어 있지만 프린터를 사용하기 전에 이를 설치해야만 합니다. 프린터 제조업체의 웹 사이트를 통해서 최신 업데이트된 드라이버를 다운로드 받을 수도 있습니다. PCL 또는 포스트스크립트 참조.

프린터 이름
적절한 프린터 이름 지정 방식에 따라 프린터 이름을 입력하십시오. 이 이름은 시스템의 프린터 폴더에서 이 프린터에 대한 프린터 아이콘을 식별하는 데 사용됩니다. 프린터 폴더를 표시하려면 Windows 바탕 화면에서 시작을 누르고 설정을 선택한 다음 프린터를 누르십시오.

ARP
Address Resolution Protocol(주소 결정 프로토콜). ARP는 IP 주소를 이더넷 주소와 같은 물리적 주소(DLC 주소)로 변환하는데 사용되는 TCP/IP 프로토콜입니다. 물리적 주소를 얻으려는 호스트는 ARP 요청을 TCP/IP 네트워크로 브로드캐스트합니다. 그러면 요청에 대한 IP 주소가 있는 네트워크 상의 호스트는 물리적 하드웨어 주소를 응답합니다.

BootP

HP Jetdirect 프린트 서버
HP Jetdirect 프린트 서버는 프린터를 네트워크에 직접 연결하는 HP 네트워크 보조 장치입니다. 네트워크 직접 연결 방식은 위치상의 신축성을 제공하며 최고의 프린터(또는 일체형 장치) 성능을 보장해 줍니다.
HP Jetdirect 내부 프린트 서버 카드는 입출력(I/O) 슬롯이 설치된 HP 프린터에 사용됩니다. 일부 HP Jetdirect 외장형 프린트 서버는 병렬 포트가 설치된 대부분의 프린터에 사용할 수 있는 반면, 기타 HP Jetdirect 프린트 서버는 USB 포트가 설치된 거의 모든 프린터에 사용할 수 있습니다. 두 가지 유형의 프린트 서버는 일반적인 네트워크 토폴로지와 프로토콜을 모두 지원합니다.

-I-

INF 파일
INF 파일은 .inf라는 확장명을 가지며 장치에 관한 정보를 보유하고 있습니다. 이 파일은 장치의 이름, 복사 대상 파일, 파일의 위치 등을 설치 프로그램에 알려 줍니다.

IP 주소
인터넷 프로토콜 주소. IP 주소는 TCP/IP 네트워크상에서 각 장치를 구분해주며 일반적으로 네트워크 관리자에 의해 할당됩니다. 최신 HP Jetdirect 프린트 서버는 IPv4와 IPv6 주소를 모두 지원합니다. Jetdirect 프린터 서버가 켜지면 네트워크 서버에서 올바른 IP 주소를 구성하려고 시도하는 것이 일반적입니다. 올바른 IP 주소를 할당할 수 없으면 기본 주소가 자동으로 할당됩니다. 복수의 장치가 동일한 기본 주소로 구성되어 있을 경우, LAN Hardware(MAC) 주소를 확인하여 정확한 장치를 식별하십시오.

IP 호스트 이름
IP 호스트 이름은 프린터의 IP 주소 대신 사용할 수 있는 사용이 간편한(또는 식별하기 쉬운) 이름입니다. 이는 네트워크 상의 이름 서버가 호스트 이름을 대응하는 IP 주소에 매핑하도록 설정된 경우에 사용할 수 있습니다. 호스트 이름이 프린터에 지정 및 구성되어진 경우(예: printer1.marketing.com) 프린터를 식별하기 위해 호스트 이름을 지정할 수 있습니다.

IPv4
IPv4는 인터넷 프로토콜의 버전 4입니다. IPv4 주소는 점으로 구분된 10진수 형식의 32비트로 구성되어 있습니다(예: 192.168.100.250). 만일 네트워크 서버가 올바른 IPv4 주소를 할당하지 않았다면 Jetdirect 프린트 서버는 네트워크에 따라 링크-로컬/자동 IP 주소(169.254.x.x) 또는 기본 레거시 주소(192.0.0.192)를 할당하게 됩니다.

IPv6
IPv6은 인터넷 프로토콜의 개선된 버전으로서, IPv4를 대체할 수도 있지만 이와 공존할 수도 있습니다. IPv6 주소는 콜론으로 구분된 16진수 형식의 128비트로 구성되어 있습니다(예: 2001:0DB8:0000:0000:0000:0000:1428:57AB). 따라서, IPv6 주소는 네트워크에 추가되는 장치의 증가를 얼마간 유지할 수 있습니다. IPv4와 달리, 프린트 서버에는 여러 개의 IPv6 주소를 할당할 수 있습니다. 기본 링크-로컬 IPv6 주소가 자동으로 할당되지만, 라우터나 서버 기반의 수동 IPv6 주소도 할당이 가능합니다.

LAN 하드웨어, MAC 또는 스테이션 주소

WEP
무선 네트워크 트래픽을 암호화하기 위해 Wi-Fi 네트워크에서 사용하는 최초 보안 표준입니다. WEP는 하나의 무선 장치에서 다른 무선 장치로 전파를 통해 데이터를 전송할 때 해당 데이터를 암호화하는 보안성을 제공함으로써 외부에서 데이터를 해석하지 못하도록 방지합니다. 프린터와 동일한 WEP 설정을 공유하는 장치만 프린터와 통신할 수 있습니다.

WEP 키
WEP 키, 즉 "암호화 키"는 영숫자 문자 또는 16진수의 조합으로 구성됩니다. WEP 키를 생성한 후에는 잘 기억해 두거나 안전한 장소에 기록해 두어야 하며, 잊어버린 경우 복구가 불가능합니다. WEP 키의 길이는 64비트 또는 128비트이며, 키의 처음 24비트는 자동으로 제공됩니다. 따라서 WEP 키를 생성할 때는 나머지 비트(64비트 키인 경우 40비트, 128비트 키인 경우에는 104비트)만 만들게 됩니다.
WEP 키를 만들려면 WEP 키가 영숫자인지 또는 16진수인지 여부와 포함된 비트 수를 고려하여 다음을 입력하십시오.
